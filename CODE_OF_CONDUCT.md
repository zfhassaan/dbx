# Code of Conduct for the DBX Database Backup Utility

This Code of Conduct outlines the expectations for all contributors participating in the **DBX Database Backup Utility** open-source project. Our goal is to maintain a welcoming, safe, and productive environment for developers, DevOps engineers, database administrators, and contributors building tools related to **database backup, restore, automation, and cloud storage workflows**.

## Our Commitment
We are committed to fostering an inclusive and respectful community, ensuring that DBX remains a professional and collaborative ecosystem for individuals working on database automation, Go-based CLI tools, and open-source infrastructure utilities.

## Expected Behavior
Positive participation includes:
- Being respectful, supportive, and constructive  
- Providing useful feedback when reviewing pull requests  
- Helping other contributors, especially newcomers  
- Collaborating professionally on issues related to database backup, restore, logging, scheduling, cloud uploads, and CLI development  

## Unacceptable Behavior
The following behaviors negatively impact the community and will not be tolerated:
- Harassment, discrimination, hate speech  
- Personal attacks or insults  
- Publishing personal information without consent  
- Derailing constructive technical discussion  
- Aggressive or disruptive communication  

## Maintainer Responsibilities
Project maintainers may:
- Remove or edit comments that violate this policy  
- Close issues or PRs that ignore guidelines  
- Ban contributors engaging in harmful behavior  

## Scope
This Code of Conduct applies to:
- GitHub Issues  
- Pull Requests  
- Discussions  
- Community forums  
- Any public interactions related to DBX
